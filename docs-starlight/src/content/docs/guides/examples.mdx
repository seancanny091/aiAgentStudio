## Customer Support Agent

Help desk agent that searches cases, updates status, and creates tasks.

### Agent Configuration

<CardGrid>
  <Card title="Conversational" icon="chat">
    **Memory**: BufferWindow · **History Limit**: 10 turns
  </Card>
  <Card title="Identity Prompt" icon="seti:text">
    Customer support assistant for the service team — finds cases, updates status, creates follow-up tasks.
  </Card>
</CardGrid>

**Instructions Prompt**:
```
- Search for cases before asking users for case numbers
- Confirm before updating case status or priority
- Create tasks for any follow-up actions mentioned
- Summarize case details clearly
```

### Capabilities

<Tabs>
  <TabItem label="Search Cases">

```json
{
  "capabilityName": "search_cases",
  "description": "Search for support cases by number, contact, account, or subject",
  "implementationType": "Standard",
  "standardActionType": "GetRecordDetails",
  "backendConfiguration": {
    "objectApiName": "Case",
    "defaultFields": ["CaseNumber", "Subject", "Status", "Priority", "Contact.Name", "Account.Name"]
  }
}
```

  </TabItem>
  <TabItem label="Update Case">

```json
{
  "capabilityName": "update_case",
  "description": "Update case status, priority, or other fields",
  "implementationType": "Standard",
  "standardActionType": "UpdateRecord",
  "requiresApproval": true,
  "backendConfiguration": {
    "objectApiName": "Case"
  }
}
```

  </TabItem>
  <TabItem label="Create Follow-up Task">

```json
{
  "capabilityName": "create_followup",
  "description": "Create a follow-up task for a case",
  "implementationType": "Standard",
  "standardActionType": "CreateRecord",
  "backendConfiguration": {
    "objectApiName": "Task",
    "defaultValues": {
      "Status": "Not Started",
      "Priority": "Normal"
    }
  }
}
```

  </TabItem>
</Tabs>

### Sample Conversation

```
User: Find cases for Acme Corp

Agent: I found 3 open cases for Acme Corp:
       1. Case #00001234 - Login issues (High Priority)
       2. Case #00001235 - Billing question (Normal)
       3. Case #00001236 - Feature request (Low)

       Would you like details on any of these?

User: Update the billing question to high priority

Agent: I'll update Case #00001235 to High priority.
       [Approval Required]
       Confirm this change?

User: Yes

Agent: ✓ Case #00001235 priority updated to High.
```

## Best Practices

<CardGrid>
  <Card title="Capability Design" icon="seti:gear">
    - **One purpose per capability** — don't combine search and update
    - **Clear descriptions** — tell the AI exactly when to use it
    - **Appropriate parameters** — only expose what's needed
    - **Enable approval** — for any data modifications
  </Card>
  <Card title="Prompt Engineering" icon="seti:text">
    - **Be specific** about the agent's role
    - **Include constraints** — what NOT to do
    - **Provide examples** of good responses
    - **Set expectations** for tone and format
  </Card>
  <Card title="Testing" icon="approve-check">
    - **Test edge cases** — empty results, errors, permissions
    - **Test with real users** — different profiles and permissions
    - **Monitor decision steps** — verify AI reasoning
    - **Iterate on prompts** — refine based on actual usage
  </Card>
</CardGrid>
